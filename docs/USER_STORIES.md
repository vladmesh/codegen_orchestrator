# User Stories

> **Актуально на**: 2026-02-11

Целевые сценарии использования платформы. Каждая user story описывает конечный результат с точки зрения пользователя.

---

## MVP Stories (Phase 1)

### US1: Свой токен Telegram бота
**Приоритет**: 🔴 Критический  
**Статус**: 🔶 В разработке (~80%)

**Как** пользователь  
**Хочу** использовать свой Telegram Bot Token  
**Чтобы** бот имел моё имя и был под моим контролем

**Сценарий**:
1. Система спрашивает: "Хотите создать токен сами?" (или пользователь сразу даёт токен)
2. Пользователь регистрирует бота через @BotFather
3. Пользователь скидывает токен
4. Система деплоит бот с этим токеном

**Acceptance Criteria**:
- [ ] Система принимает токен от пользователя
- [ ] Токен сохраняется как секрет проекта
- [ ] Деплой использует пользовательский токен

---

### US2: Создание бота по запросу (Auto-provisioning)
**Приоритет**: � Высокий (Phase 2)  
**Статус**: ⬜ Не начато (блокировано отсутствием пула токенов)

**Как** пользователь  
**Хочу** описать словами какой бот мне нужен  
**Чтобы** получить работающий бот через 10-15 минут (без регистрации в BotFather)

**Сценарий**:
1. Пользователь пишет: "Сделай бота погоды"
2. Система выделяет токен из пула (или регистрирует сама)
3. Система генерирует код, тестирует, деплоит
4. Пользователь получает ссылку на готового бота

**Acceptance Criteria**:
- [ ] Система имеет пул свободных токенов
- [ ] Проект создаётся автоматически с выделенным токеном
- [ ] Пользователь не взаимодействует с BotFather

## Post-MVP Stories (Phase 2+)

### US3: Доработка существующего проекта
**Приоритет**: 🟡 Высокий  
**Статус**: 🔶 Частично (copier update реализован)

**Как** пользователь с существующим ботом  
**Хочу** попросить добавить функционал  
**Чтобы** не создавать проект заново

**Сценарий**:
1. Пользователь: "Хочу чтобы мой бот погоды ещё присылал картинки котов"
2. Система понимает какой проект имеется в виду
3. Система добавляет функционал, тестирует, деплоит

---

### US4: Статус проектов
**Приоритет**: 🟡 Высокий  
**Статус**: ⬜ Не начато

**Как** пользователь с несколькими проектами  
**Хочу** спросить "как дела у моих проектов"  
**Чтобы** получить отчёт о здоровье и нагрузке

---

### US5: Автоматическое восстановление
**Приоритет**: 🟢 Средний  
**Статус**: ⬜ Не начато

**Как** пользователь  
**Хочу** получать уведомления если проект упал  
И **чтобы** система автоматически пыталась починить

---

### US6: Создание сайта (визуальное)
**Приоритет**: 🟢 Средний  
**Статус**: ⬜ Не начато

**Как** пользователь без технических знаний  
**Хочу** описать сайт словами и скриншотами  
**Чтобы** получить работающий сайт с доменом

---

### US7: Редактирование интерфейса через скриншоты
**Приоритет**: 🔵 Низкий  
**Статус**: ⬜ Не начато

**Как** пользователь  
**Хочу** обвести на скриншоте что изменить  
**Чтобы** система поняла и внесла изменения

## Admin Stories

### US-Admin: Полная видимость системы
**Приоритет**: 🔴 Критический (для Phase 2)  
**Статус**: ⬜ Не начато

**Как** администратор  
**Хочу** видеть всех пользователей, проекты, воркеров, расход токенов  
**Чтобы** контролировать систему и оптимизировать расходы

**Scope**:
- Все активные агенты и их состояние
- Логи каждого микросервиса
- Расход токенов per user/project
- Возможность менять агента для задачи
