# Внутренние токены и Managed Integrations
**Статус:** Draft
**Смысл:** Попытка избавить нетехнических пользователей от необходимости заводить собственные API-ключи (OpenAI, Stripe, почта и т.д.), предоставляя "коробочные" интеграции за внутренние токены платформы.

## Суть идеи (Managed Integrations / Credit-based Proxying)
Платформа выступает API-шлюзом (Gateway). Вместо ввода сторонних ключей, юзеры покупают внутренние токены платформы и тратят их на вызовы провайдеров. Сами вызовы совершаются через наши Enterprise ключи (мастер-аккаунты). Пользователь освобождается от необходимости уходить к другим провайдерам, заводить аккаунты, вводить банковские карты и копировать непонятные строчки ключей.

**Примеры на рынке:** Zapier, Make (Integromat), OpenRouter, Cursor, Bubble.

## Подводные камни и бизнес-риски

1. **Карточный фрод (Card Testing) и чарджбеки (Самый высокий риск)**
   Злоумышленники покупают внутренние токены с краденых кредиток -> тратят наши реальные ключи (например, парсят гигабайты текстов через OpenAI) -> настоящий владелец карты замечает транзакцию и делает chargeback -> мы в двойном минусе: платим штраф за чарджбек банку/Stripe ($15) и уже заплатили провайдеру (OpenAI) за генерацию.
2. **Бан общего "Мастер-ключа" платформы**
   Если конечный юзер захочет сгенерировать запрещенный контент (NSFW, вредоносный код), провайдер API (например, Anthropic) забанит *наш* мастер-ключ. Продакшен ляжет для всех честных пользователей.
   *Решение:* Прокидывать ID конечного юзера при каждом вызове (например, через параметр `user` в OpenAI) и оперативно банить подозрительных.
3. **Shared Rate Limits ("Проблема шумного соседа")**
   Один пользователь, запустивший тяжелую автоматизацию, может исчерпать квоты (запросы в минуту/секунду) нашего мастер-аккаунта. Другие пользователи платформы начнут получать ошибки `429 Too Many Requests`.
   *Решение:* Разработка системы очередей исполнения и локального per-tenant Rate Limiting поверх лимитов провайдеров.
4. **Математика биллинга (Приведение к единой валюте)**
   Проблема унификации тарификации: OpenAI считает текстовые токены, карты Google Maps считают показы-тайлы, базы данных — хиты/строки, хранилище — гигабайты. Нам нужно пересчитать все это в единую "внутреннюю монету".
5. **Слишком низкая маржа**
   Брать комиссию в размере 0.5% (о которой мы говорили ранее) невозможно. Платёжные провайдеры (Stripe) откусывают ~2.9% + $0.30 с каждой транзакции пополнения баланса. Наценка на такие "коробочные" интеграции должна составлять от 20% до 100%, чтобы покрыть затраты на фрод и процессинг, либо должна продаваться пакетами/фиксированными подписками.

## Топ-10 востребованных "коробочных" интеграций

Нетехнические пользователи мыслят не категориями "API" и "Синдикация", а категориями продуктовых "Фичей". Для них это просто тумблеры, которые они активируют в своем проекте.

### Относительно безопасные (Зеленая зона)
Эти фичи несут самую высокую ценность для создания продуктов с относительно контролируемым риском фрода:
1. **Текстовая магия ИИ (OpenAI / Anthropic)**
   * **Продажа:** "Умный чат-бот", "Улучшение текста", "Анализатор отзывов".
2. **Доступ продукта в интернет (Tavily / Firecrawl / SerpAPI)**
   * **Продажа:** "Искать по ссылкам", "Агент-ридер новостей". (Тарификация за запрос или страницу).
3. **Файловое хранилище (AWS S3 / Cloudinary)**
   * **Продажа:** "Диск для фотографий пользователей".
   * *Важно:* Обязательно сканировать на NSFW.
4. **Синтез и распознавание речи (ElevenLabs / Whisper)**
   * **Продажа:** "Читать текст вслух", "Голосовые сообщения в текст" (пересчитывается в секунды аудио).
5. **Интеграция с Базами (Google Sheets / Airtable)**
   * **Продажа:** "Сохранять лиды в Excel". Само API бесплатно для нас, юзеры доплачивают несколько токенов просто за то, что вы предоставляете им удобный OAuth и абстракцию без головной боли.
6. **Машинный перевод (DeepL)**
   * **Продажа:** "Авто-перевод интерфейса и сообщений".
7. **Карты и геопозиция (Mapbox / Google Maps)**
   * **Продажа:** "Автокомплит адреса", "Карта курьера".

### Критичные (Красная зона - Опасность финансового краха!)
Требуют мощных защит от ботов и жесточайших лимитов для новых аккаунтов. Без них можно обанкротиться за одну ночь.
8. **Транзакционные Email (Resend / SendGrid)**
   * **Зачем нужно:** База (подтверждение почты, сброс пароля). Письма идут от технического домена вроде `mail.your-app.com`.
   * **Риск:** Огромный риск попадания в спам-листы. Если нас забанят за спам — письма перестанут приходить у всех пользователей на платформе.
9. **Отправка SMS и WhatsApp (Twilio)**
   * **Зачем нужно:** Вход по номеру (OTP-коды) и уведомления клиентов.
   * **Риск:** SMS Pumping Fraud на платные номера стран Африки и Азии. Способны слить бюджет компании (десятки тысяч долларов) за несколько часов работы бота.
10. **Нейро-Картинки (DALL-E 3 / SD / Midjourney)**
    * **Зачем нужно:** Генераторы аватарок, баннеров.
    * **Риск:** Распространение дипфейков, нарушение авторских прав, генерация порнографии.

## Архитектурные требования (High level)
Для реализации нужна отдельная инфраструктура: **Proxy & Billing Gateway**.
Его ответственность:
- Перехватывать все исходящие запросы от сгенерированного юзерского приложения к "Магическим фичам".
- Перед отправкой запроса провайдеру (до проксирования!), проверять баланс "токенов" клиента (быстрый Redis-кеш).
- Считать размер потребления, отправлять запрос, дожидаться ответа, списывать актуальное количество потраченных токенов.
- Блокировать юзера по лимитам за спам/перерасход (Circuit Breaker).
