## Problems Found

### Problem 1: Host venv shebang incompatible with container runtime

- **Type**: template
- **Severity**: critical
- **Description**: Integration tests fail with `cannot execute: required file not found` when running `alembic` in the backend container.
- **Root cause**: The template's `compose.tests.integration.yml.jinja` (line 17) sets `PATH: /workspace/services/backend/.venv/bin:...`. In CI, the flow is:
  1. `make setup` runs on the GitHub Actions host → creates `services/backend/.venv` with shebangs pointing to host Python (`#!/opt/hostedtoolcache/Python/3.12.12/x64/bin/python`)
  2. Docker Compose builds the image (venv inside at `/app/services/backend/.venv`)
  3. Volume mount `../:/workspace:delegated` maps the **host repo** (including host-built `.venv`) into the container
  4. PATH points to `/workspace/services/backend/.venv/bin` → finds host's `alembic` binary → shebang references non-existent host Python path inside container
- **Template file**: `/home/vlad/projects/service-template/template/infra/compose.tests.integration.yml.jinja`, lines 17 and 48
- **Suggested fix**: Change PATH to use the Docker image's venv (`/app/services/backend/.venv/bin`), or add anonymous volume to shadow the host venv:
  ```yaml
  # Option A: use image venv for binaries
  PATH: /app/services/backend/.venv/bin:/usr/local/bin:...

  # Option B: shadow host venv with image venv
  volumes:
    - ../:/workspace:delegated
    - /workspace/services/backend/.venv  # prevents host venv from being mounted
  ```

### Problem 2: Backend service has no readiness check in integration compose

- **Type**: template
- **Severity**: major
- **Description**: After fixing the venv PATH (commit `9fc590d`), CI fails with `httpx.ConnectError: All connection attempts failed`. Integration tests start immediately when backend container starts, without waiting for migrations + uvicorn to be ready.
- **Root cause**: `compose.tests.integration.yml.jinja` line 50-51 uses `condition: service_started` instead of a healthcheck. The backend needs ~5-10 seconds to run migrations and start uvicorn, but tests begin immediately.
- **Additional factor**: Backend container produced **zero output** in CI logs despite running for 14 seconds. This suggests the backend itself may have crashed silently (possibly another PATH/import issue) or output buffering hides the error.
- **Suggested fix**: Add healthcheck to backend in integration compose:
  ```yaml
  backend:
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 2s
      timeout: 3s
      retries: 15
    ...
  integration-tests:
    depends_on:
      backend:
        condition: service_healthy
  ```
