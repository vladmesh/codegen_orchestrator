## Найденные баги

### Bug 1: `TaskCreate` schema не принимает `status` (CRITICAL) — ✅ FIXED

- **Сервис**: api
- **Файл**: `services/api/src/schemas/task.py` — class `TaskCreate`
- **Суть**: Pydantic schema `TaskCreate` не содержит поле `status`. Architect tools отправляет `"status": "todo"`, но API его молча отбрасывает. SQLAlchemy модель `Task` использует `default=TaskStatus.BACKLOG`. В итоге все задачи создаются как `backlog`.
- **Эффект**: Scheduler поллит `status=todo` и ничего не находит → pipeline мёртв после architect.
- **Фикс**: Добавлено `status: TaskStatus = TaskStatus.BACKLOG` в `TaskCreate` + заменён хардкод в router (`create_task`, `push_task`). API перезапущен.

### Bug 2: PO не создаёт Repository (CRITICAL) — ✅ FIXED

- **Сервис**: langgraph (PO tools)
- **Суть**: PO создаёт Project и Story, но не создаёт Repository. Scheduler'овский `scaffold_trigger.py` проверяет наличие repository (строка 57-59) и пропускает проект если нет.
- **Эффект**: Даже если scaffolder будет запущен — trigger не сработает.
- **Связанная задача**: `task-29cbc5ec` — "Integrate Repository into production flows" (backlog, prio 6)
- **Фикс**: Добавлен вызов `POST /api/repositories/` в `create_project` с placeholder `git_url`. Langgraph перезапущен.

### Bug 3: Scaffolder контейнер не запущен (CRITICAL) — ✅ FIXED

- **Суть**: `docker ps -a | grep scaff` — пусто. Сервис определён в `docker-compose.yml` но не билдился/запускался.
- **Эффект**: Даже при корректном trigger — некому обработать `scaffold:queue`.
- **Фикс**: `docker compose up -d --build scaffolder` — сбилдился и запущен.

### Bug 4: Architect создаёт инфраструктурные задачи (MEDIUM)

- **Суть**: Из-за Bug 3 (scaffolder не работает), project.config.tree = None. Architect не видит что scaffolding уже генерирует базовую инфраструктуру и создаёт задачу "Set up Telegram bot infrastructure" — вещи, которые делает scaffolder.
- **Root cause**: Отсутствие tree в project config. Architect промпт говорит "Never create tasks for Docker, compose, CI, deployment — scaffolding handles it", но без tree он не знает, что bot setup тоже scaffolded.
- **Эффект**: Лишняя задача, worker будет делать работу которая уже сделана.
- **Фикс**: Исправить bugs 2+3 → scaffolder заполнит tree → architect увидит структуру.
