### Problem 2: Server IP not resolved from port allocations API
- **Type**: meta
- **Severity**: minor
- **Backlog**: `--`
- **Description**: The port allocations API (`/api/servers/{handle}/ports`) returns records but the `server_ip` field is empty, causing the server verification script to fail with "Could not resolve hostname". Had to use the deployed_url from the deploy task result instead.

- **Suggested fix**: Update the E2E skill to extract server IP from `deployed_url` as primary method, or fix the API to include `server_ip` in port allocation responses.
