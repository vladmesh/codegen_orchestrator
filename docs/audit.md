# Аудит архитектуры

> **Дата**: 2026-02-10
> **Контекст**: Результаты ревизии документации и кода

---

## 🔴 Критические проблемы

### 1. TesterNode — заглушка
**Где**: `services/langgraph/src/nodes/` (Engineering Subgraph)
**Проблема**: TesterNode всегда возвращает `passed=True`. Реального запуска тестов не происходит.
**Влияние**: Код от Developer не проверяется перед деплоем.
**Рекомендация**: Реализовать запуск тестов через worker-manager (аналогично Developer).

### 2. API Authentication отсутствует
**Где**: `services/api/`
**Проблема**: API endpoints защищены только `x-telegram-id` header, нет полноценной аутентификации.
**Влияние**: Любой с доступом к сети может вызывать API.
**Рекомендация**: Добавить API key / JWT аутентификацию.

---

## 🟡 Средние проблемы

### 3. Resource Limits не реализованы (Worker Manager)
**Где**: `services/worker-manager/`
**Проблема**: Нет ограничений на:
- `MAX_CONCURRENT_WORKERS` — количество одновременных контейнеров
- Memory/CPU limits на контейнеры
- Disk usage limits
**Влияние**: Один пользователь может исчерпать все ресурсы хоста.
**Рекомендация**: Добавить конфигурируемые лимиты в worker-manager.

### 4. Idle Pause/Wakeup не реализован
**Где**: `services/worker-manager/`
**Проблема**: PO Worker контейнеры не приостанавливаются при бездействии.
**Влияние**: Ресурсы расходуются на простаивающие контейнеры.
**Рекомендация**: Реализовать `docker pause`/`unpause` по таймауту неактивности.

### 5. Creation Queue не реализована
**Где**: `services/worker-manager/`
**Проблема**: Нет очереди создания воркеров — каждый запрос создаёт контейнер немедленно.
**Влияние**: При пиковой нагрузке все запросы стартуют одновременно.
**Рекомендация**: Добавить очередь и управление приоритетами.

### 6. Deploy-worker и engineering-worker — неявные сервисы
**Где**: `docker-compose.yml`
**Проблема**: `deploy-worker` и `engineering-worker` фактически — отдельные инстансы `langgraph` с другим `command`. В документации они не всегда упоминаются отдельно.
**Рекомендация**: Явно документировать как worker-процессы сервиса langgraph.

## 🟢 Низкоприоритетные

### 7. Analyst и Zavhoz — мёртвый код
**Где**: `services/langgraph/src/nodes/analyst.py`, `nodes/zavhoz.py`
**Проблема**: Код есть, но ноды не подключены в граф. Документируются как "не используется".
**Рекомендация**: Удалить или задокументировать как Phase 3.

### 8. Developer Queue Routing отличается от спецификации
**Где**: `services/worker-manager/`
**Проблема**: Спецификация описывала per-worker unique queues, реализация использует shared queue.
**Влияние**: Минимальное — текущая реализация работает корректно.

### 9. Backlog содержит устаревшие задачи
**Где**: `docs/backlog.md`
**Проблема**: Ряд задач уже выполнен или стал неактуальным, но не отмечен.
**Рекомендация**: Регулярная ревизия backlog.

### 10. E2E тесты не завершены
**Где**: `tests/e2e/`
**Проблема**: Фазы 5-7 E2E Engineering Test Plan не реализованы. Full system docker-compose validation не пройдена.
**Рекомендация**: Завершить E2E покрытие перед production release.
